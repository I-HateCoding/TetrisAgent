# 基于 After-state DQN 的俄罗斯方块智能体训练与优化

## 摘要

本项目基于 `tetris_gymnasium` 环境训练俄罗斯方块智能体，目标是在给定当前棋盘和下一个方块的情况下，选择较优的落子方式以最大化累计 reward。项目首先实现了基于 DQN 的基础训练流程，随后针对训练过程中出现的策略不稳定、不同随机种子表现差异较大、偶发极端低分等问题，在不改变 Q-learning/DQN 主框架的前提下，对状态表示和训练流程进行了优化。

最终模型采用 after-state DQN：环境为每个合法动作生成落子后的候选棋盘，神经网络对这些候选棋盘分别估值，并选择价值最高的动作。目前训练结果显示，模型已经能够在多数局中取得几千到上万的 reward，少数随机种子下仍会出现几百分的极端低分，说明模型已经具备一定策略能力，但泛化稳定性仍有提升空间。

## 1. 任务与环境

俄罗斯方块是一个典型的序列决策问题。智能体在每一步需要根据当前棋盘、当前方块以及后续局面风险选择一个落子动作。若只追求当前一步的消行收益，可能会导致棋盘结构恶化，例如产生较多空洞或堆叠过高；因此智能体需要学习一种能够兼顾即时 reward 和长期生存能力的策略。

本项目使用 `tetris_gymnasium` 作为训练环境，并使用 `GroupedActionsObservations` 包装环境。包装后，动作空间不再表示单步移动或旋转，而是表示当前方块的一种最终落子方案。对于每个 action，环境会在 observation 中给出执行该动作后的候选棋盘：

```text
observation[action] = 执行该落子动作后的棋盘
```

因此，本项目可以直接比较不同落子后的棋盘质量，而不需要让智能体逐步学习左移、右移、旋转和硬降等低层操作。这种建模方式降低了动作序列搜索难度，更适合使用 after-state value network。

本项目中的主要指标为环境返回的累计 reward。训练日志中的 `raw` 表示一局游戏中环境原始 reward 的累计值；`shaped` 表示用于训练的辅助 reward 信号。实际评估时主要关注 `raw reward`。

## 2. 方法：Q-learning 与 After-state DQN

### 2.1 Q-learning 基本公式

Q-learning 的目标是学习状态-动作价值函数 `Q(s, a)`，表示在状态 `s` 下执行动作 `a` 后，未来能够获得的期望累计回报。其经典更新公式为：

```text
Q(s, a) <- Q(s, a) + alpha [r + gamma max_a' Q(s', a') - Q(s, a)]
```

其中：

- `s` 为当前状态；
- `a` 为当前动作；
- `r` 为执行动作后获得的即时 reward；
- `s'` 为下一状态；
- `alpha` 为学习率；
- `gamma` 为折扣因子；
- `max_a' Q(s', a')` 表示下一状态下最优动作的估计价值。

当状态空间较大时，无法直接用表格保存所有 `Q(s, a)`。DQN 使用神经网络近似 Q 函数：

```text
Q(s, a; theta)
```

训练时构造目标值：

```text
y = r + gamma max_a' Q(s', a'; theta-)
```

并最小化预测值与目标值之间的误差：

```text
L(theta) = E[(y - Q(s, a; theta))^2]
```

其中 `theta` 是当前网络参数，`theta-` 是 target network 参数，用于提高训练稳定性。

### 2.2 Double DQN

基础 DQN 中，动作选择和动作估值都由 target network 完成，可能造成 Q 值过估计。本项目训练脚本采用 Double DQN 思路：用当前网络选择下一步动作，用 target network 估计该动作价值：

```text
a* = argmax_a' Q(s', a'; theta)
```

```text
y = r + gamma Q(s', a*; theta-)
```

这样可以减少价值估计过高的问题，使训练更加稳定。

### 2.3 After-state DQN

由于环境已经为每个动作生成了落子后的棋盘，本项目没有直接学习 `Q(s, a)`，而是学习落子后棋盘的价值函数。设 `b_a` 表示在当前状态下执行动作 `a` 后得到的棋盘，则网络估计：

```text
V(b_a; theta)
```

执行动作时，对所有合法动作对应的候选棋盘分别打分：

```text
a* = argmax_a V(b_a; theta)
```

这种方式被称为 after-state value learning。它的优点是输入更直接：网络只需要判断“这个落子后的棋盘好不好”，而不需要同时处理原状态和动作编码。

本项目最初使用 CNN 直接输入候选棋盘，输出一个标量价值。后续优化中，在棋盘图像之外额外加入了少量棋盘结构特征，例如洞数、高度、凹凸度等，以帮助网络更稳定地判断棋盘质量。

## 3. 初始训练与问题观察

基础版本完成了 DQN 的训练、保存、加载和评估流程。训练时使用经验回放 replay buffer、target network、epsilon-greedy 探索和 checkpoint 保存机制。初始训练能够让智能体学到基本落子策略，但也观察到以下问题：

1. 训练 reward 波动较大。某些 episode 可以取得较高分数，但也存在快速失败的局。
2. 不同随机种子下表现差异明显，说明模型对某些方块序列或局面分布适应较快，但泛化稳定性不足。
3. 纯 CNN 输入有时会学到局部策略，例如短期内看似得分较高，但会留下较多空洞，影响长期生存。
4. 训练日志中的高分 episode 不一定代表模型整体泛化能力强，因此需要多 seed 评估。

目前观察到的评估分布大致为：少数局会出现几百 reward 的极端低分；多数局 reward 位于几千到上万之间；部分较好的局可以达到更高分数。这说明模型已经具备有效策略，但仍存在不稳定失败的情况。

## 4. 优化策略

本项目的优化没有更换核心算法，仍然保持 DQN/Q-learning 框架。优化重点是改善状态表示和训练稳定性。

### 4.1 棋盘特征增强

纯 CNN 需要从棋盘图像中自行学习“洞数多不好”“堆叠过高危险”“表面过于凹凸会影响后续落子”等结构规律。为了降低学习难度，本项目在棋盘图像之外加入了人工棋盘特征：

- 聚合高度；
- 最大列高度；
- 洞数；
- 相邻列高度差，即凹凸度；
- 平均高度；
- 高度标准差；
- 井深；
- 空列比例。

最终网络输入可表示为：

```text
x = [board, features(board)]
```

该优化并没有改变 DQN 的目标函数，只是改善了价值函数的输入表示，使网络更容易识别对俄罗斯方块长期生存重要的棋盘结构。

### 4.2 Reward shaping

环境原始 reward 作为最终评估指标，但训练时引入了 shaped reward 辅助学习。其思想是奖励棋盘结构变好、惩罚棋盘结构变差，例如：

- 洞数增加时扣分；
- 洞数减少时加分；
- 高度或凹凸度增加时扣分；
- 游戏结束时给予额外惩罚。

这样做的目的不是改变最终评价目标，而是为 DQN 提供更密集的学习信号，使其更快理解哪些落子会造成长期风险。

### 4.3 训练稳定性优化

训练过程中还观察到一个问题：俄罗斯方块 episode 长度不固定。一个高分 episode 可能包含数千甚至上万步，这会向 replay buffer 中加入大量相似样本，并触发大量梯度更新。因此，模型可能在短时间内偏向某个随机种子或某类局面。

为减小这种影响，后续训练采用了更保守的设置：

- 降低学习率；
- 减少训练更新频率，例如每多步训练一次；
- 增大 replay buffer；
- 使用固定 validation seeds 评估 checkpoint；
- 保存 `eval_p25` 表现较好的 checkpoint，而不仅仅看平均 reward。

其中 `eval_p25` 表示评估分数的 25 分位数。相比平均分，它对少数高分局不那么敏感，更能反映模型是否稳定、是否容易暴毙。

## 5. 实验结果

本节结果待最终训练完成后填充。建议报告中同时展示平均表现和稳定性指标，而不是只展示最高分。

### 5.3 结果分析

从训练过程看，模型已经不再只是随机落子，而是能够在多数情况下维持较长局面并获得较高 reward。高分 episode 表明 after-state DQN 能够学习到有效的棋盘价值判断。然而，多 seed 评估中仍然存在极端低分，说明模型对某些开局或方块序列仍不够鲁棒。

加入棋盘特征后，模型对洞数、高度、凹凸度等关键因素的判断更加稳定，坏习惯逐渐减少。使用固定 validation seeds 和 `eval_p25` 选择 best checkpoint 后，模型选择标准也从“偶尔高分”转向“整体不容易失败”。

## 6. 总结

本项目实现了一个基于 after-state DQN 的俄罗斯方块智能体。首先，利用 `GroupedActionsObservations` 将动作建模为最终落子方案，使网络可以直接评价每个候选落子后的棋盘。其次，基于 Q-learning/DQN 的价值更新公式，训练神经网络近似棋盘价值函数，并使用 Double DQN、replay buffer 和 target network 提高训练稳定性。

在基础模型训练后，项目观察到策略不稳定和不同随机种子表现差异较大的问题。随后，在不改变 DQN 主框架的前提下，通过加入棋盘结构特征、使用 shaped reward、降低训练更新强度、固定 validation seeds 和采用 `eval_p25` 保存 checkpoint 等方式，逐步改善模型稳定性。

目前结果表明，模型已经能够在多数 episode 中取得几千到上万的 reward，说明其具备一定的有效落子策略；但少数极端低分仍然存在，后续可以继续围绕泛化稳定性进行改进，例如进一步扩大评估种子数量、优化 replay buffer 采样方式或调整网络结构。
